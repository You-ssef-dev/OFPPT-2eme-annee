⭐ 1. Introduction : À quoi servent procédures & fonctions ?

Les deux permettent de stocker du code MySQL réutilisable.

Élément Sert à Peut retourner S'utilise dans
Procédure stockée Automatiser une tâche Rien (mais peut renvoyer OUT) Appel direct CALL
Fonction Calculer et retourner une valeur Obligatoirement 1 valeur Dans un SELECT
⭐ 2. Les blocs de code

Une procédure ou une fonction commence toujours par un bloc :

SQL
BEGIN
    -- instructions SQL
END;

Pour écrire plusieurs instructions, il faut changer le délimiteur :

SQL
DELIMITER $$

... code ...

DELIMITER ;
⭐ 3. Les fonctions : syntaxe simple

Exemple débutant : fonction qui retourne le nombre de clients dans une ville

SQL
DELIMITER $$

CREATE FUNCTION nb_clients(ville VARCHAR(50))
RETURNS INT
BEGIN
    RETURN (SELECT COUNT(*) FROM clients WHERE adresse = ville);
END $$

DELIMITER ;

Usage :

SQL
SELECT nb_clients('Agadir');

Une fonction doit toujours :

  • ✔ retourner une valeur
  • ✔ déclarer son type de retour (RETURNS ...)
  • ✔ être utilisée dans un SELECT ou une expression
⭐ 4. Les procédures stockées

Exemple simple : afficher les clients d’une ville

SQL
DELIMITER $$

CREATE PROCEDURE clients_par_ville(IN v VARCHAR(50))
BEGIN
    SELECT * FROM clients WHERE UPPER(adresse) = UPPER(v);
END $$

DELIMITER ;

Usage :

SQL
CALL clients_par_ville('Agadir');
⭐ 5. Les paramètres IN / OUT / INOUT
Type Direction Explication
IN Entrée On donne une valeur
OUT Sortie La procédure retourne une valeur
INOUT Entrée + sortie La valeur peut être modifiée

Exemples rapides :

SQL
-- IN :
(IN ville VARCHAR(50))

-- OUT :
(OUT total INT)

-- Appel :
CALL procedure(..., @sortie);
SELECT @sortie;

-- INOUT :
(INOUT compteur INT)
⭐ 6. Supprimer une procédure ou fonction
SQL
DROP PROCEDURE nom_procedure;
DROP FUNCTION nom_fonction;

Afficher les procédures existantes :

SQL
SHOW PROCEDURE STATUS;
⭐ 7. Pourquoi les utiliser ?
  • ✔ réutilisation du code
  • ✔ sécurité (limite les injections SQL)
  • ✔ performance (moins de trafic réseau)
  • ✔ organisation du code métier
  • ✔ automation de tâches

Expert 🟪 PARTIE B — NIVEAU EXPERT

⭐ 1. Les déclarations informatives (caractéristiques des fonctions)

Une fonction peut avoir des options :

Option Signification
DETERMINISTIC Donne toujours la même sortie si les arguments sont les mêmes
READS SQL DATA Peut lire la base
MODIFIES SQL DATA Peut modifier la base (rare, déconseillé dans une fonction)
CONTAINS SQL Contient du SQL mais ne lit/modifie rien

Exemple :

SQL
RETURNS INT
DETERMINISTIC
⭐ 2. Les procédures avancées

Les procédures peuvent :

  • ✔ contenir IF, CASE, LOOP, WHILE, REPEAT
  • ✔ appeler d’autres procédures
  • ✔ utiliser des curseurs
  • ✔ gérer des exceptions

Exemple complet avec OUT :

SQL
DELIMITER $$

CREATE PROCEDURE nb_clients_par_ville(
    IN ville VARCHAR(50),
    OUT total INT
)
BEGIN
    SELECT COUNT(*) INTO total
    FROM clients
    WHERE UPPER(adresse) = UPPER(ville);
END $$

DELIMITER ;

Usage :

SQL
CALL nb_clients_par_ville('Agadir', @t);
SELECT @t;
⭐ 3. Paramètre INOUT (rare mais très puissant)
SQL
DELIMITER $$

CREATE PROCEDURE incrementer(INOUT compteur INT, IN inc INT)
BEGIN
    SET compteur = compteur + inc;
END $$

DELIMITER ;

Usage :

SQL
SET @c = 1;
CALL incrementer(@c, 5);  -- @c devient 6
⭐ 4. Gestion d’erreurs dans une procédure (HANDLERS)

Exemple : gérer erreur NOT FOUND

SQL
DECLARE EXIT HANDLER FOR NOT FOUND
    SET msg = 'Aucun client trouvé';

Exemple : gérer une erreur SQL générale

SQL
DECLARE CONTINUE HANDLER FOR SQLEXCEPTION
    SET erreur = 1;
⭐ 5. Lever une exception (SIGNAL)

Très utile pour bloquer une action interdite.

Exemple : interdire une commande inexistante :

SQL
IF nb = 0 THEN
    SIGNAL SQLSTATE '45000'
    SET MESSAGE_TEXT = 'Commande introuvable';
END IF;
⭐ 6. Relever l’erreur depuis un handler (RESIGNAL)
SQL
DECLARE CONTINUE HANDLER FOR division_by_zero
    RESIGNAL SET MESSAGE_TEXT = 'Division impossible';
⭐ 7. Meilleure structure d’une procédure professionnelle

Ordre recommandé :

  1. déclaration des variables
  2. déclaration des exceptions
  3. ouverture des curseurs (si besoin)
  4. logique métier
  5. validation ou rollback (si transactions)
⭐ 8. Générer une procédure propre et maintenable

Voici un modèle :

SQL
DELIMITER $$

CREATE PROCEDURE my_procedure(IN p_id INT, OUT message VARCHAR(255))
BEGIN
    DECLARE error_flag INT DEFAULT 0;

    DECLARE EXIT HANDLER FOR SQLEXCEPTION
    BEGIN
        SET error_flag = 1;
        RESIGNAL;
    END;

    START TRANSACTION;

    UPDATE clients SET adresse='Test' WHERE id = p_id;

    IF error_flag = 0 THEN
        COMMIT;
        SET message = 'OK';
    ELSE
        ROLLBACK;
        SET message = 'Erreur';
    END IF;

END $$

DELIMITER ;
