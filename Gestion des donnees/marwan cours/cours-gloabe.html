🔵 PARTIE I — LES BASES RELATIONNELLES & MySQL

1️⃣ Introduction : C'est quoi un SGBD relationnel ?

Un SGBD relationnel stocke les données sous forme de tables (lignes + colonnes).
Ce type de base utilise un schéma fixe, ce qui garantit :

  • cohérence des données
  • relations claires entre tables (PK/FK)
  • transactions fiables (ACID)
📌 Exemples : MySQL, PostgreSQL, Oracle, SQL Server.

2️⃣ Le langage SQL (Structured Query Language)

⭐ Trois grandes catégories :

🔹 DDL : Data Definition Language

Créer / modifier la structure de la base.

Action Exemple
Créer une base CREATE DATABASE shop;
Créer une table CREATE TABLE clients (...);
Modifier ALTER TABLE clients ADD email VARCHAR(200);
Supprimer DROP TABLE clients;

🔹 DML : Data Manipulation Language

Insérer, modifier, supprimer des données.

INSERT INTO clients(nom, prenom) VALUES("Ali","Said");

UPDATE clients SET adresse="Agadir" WHERE id=1;

DELETE FROM clients WHERE id=10;

🔹 DQL : Data Query Language

Interroger les données : SELECT

Exemples :

SELECT * FROM clients;

SELECT nom, prenom FROM clients WHERE adresse="Agadir";

3️⃣ Les Vues & Tables Temporaire

🔹 Vue = une requête enregistrée comme une "table virtuelle"

CREATE VIEW vw_clients_agadir AS
SELECT * FROM clients WHERE adresse="Agadir";

🔹 Table temporaire

Utilisée pour stocker un résultat temporaire :

CREATE TEMPORARY TABLE temp_clients
SELECT * FROM clients WHERE adresse="Agadir";

4️⃣ Programmation procédurale MySQL

MySQL permet d'écrire du code "comme un langage de programmation" :

  • variables
  • conditions
  • boucles
  • procédures stockées
  • fonctions
  • déclencheurs

5️⃣ Les Structures de contrôle

🔹 IF – ELSE

IF age >= 18 THEN
  SET categorie = "Adulte";
ELSE
  SET categorie = "Mineur";
END IF;

🔹 CASE (équivalent switch)

CASE pays
  WHEN 'Maroc' THEN SET taxe = 20;
  WHEN 'USA' THEN SET taxe = 10;
  ELSE SET taxe = 15;
END CASE;

🔹 Boucles : LOOP, WHILE, REPEAT

Exemple WHILE :

WHILE i < 10 DO
  SET i = i + 1;
END WHILE;

6️⃣ Transactions (BEGIN, COMMIT, ROLLBACK)

Permettent d'assurer l'intégrité d'une suite d'opérations :

📌 Exemple : transfert bancaire 
START TRANSACTION;

UPDATE compte SET solde = solde - 200 WHERE id=1;
UPDATE compte SET solde = solde + 200 WHERE id=2;

COMMIT;

Si erreur → ROLLBACK.

7️⃣ Les exceptions (Handlers)

Exemple : empêcher l'insertion d'un nom NULL

DECLARE EXIT HANDLER FOR 1048
SET message = "Le nom est obligatoire";

8️⃣ Les curseurs (Cursors)

Utiles pour parcourir ligne par ligne un résultat SELECT.

Exemple de boucle sur un curseur :

DECLARE cur CURSOR FOR SELECT nom FROM clients;

OPEN cur;
FETCH cur INTO varNom;
...
CLOSE cur;

9️⃣ Les Triggers (déclencheurs)

S'exécutent automatiquement lors d'un INSERT, UPDATE, DELETE.

Exemple : empêcher l'ajout d'un client de moins de 18 ans :

CREATE TRIGGER avant_insert_client
BEFORE INSERT ON clients
FOR EACH ROW
BEGIN
  IF NEW.age < 18 THEN
    SIGNAL SQLSTATE '50001'
    SET MESSAGE_TEXT = 'Client mineur interdit';
  END IF;
END;

🔟 Optimisation de MySQL

Conseils :

  • éviter SELECT *
  • créer des INDEX sur les colonnes fréquemment utilisées
  • optimiser les requêtes avec EXPLAIN
  • éviter LIKE '%mot%'
  • préférer EXISTS à IN

Exemple d'index :

CREATE INDEX idx_clients_adresse ON clients(adresse);

1️⃣1️⃣ Sécurisation MySQL

🔐 Création utilisateur :

🟣 PARTIE II — BASES NoSQL & MongoDB

Contenu basé sur le fichier OFPPT Partie 2

1️⃣ Pourquoi NoSQL ?

Les BD relationnelles sont limitées pour le Big Data :

  • énormes volumes
  • rapidité
  • données non structurées
NoSQL = Not Only SQL
→ plus flexible, distribué, adapté au web moderne.

2️⃣ Types de bases NoSQL

Type Description Exemples
Clé-Valeur Données simples, très rapides Redis, DynamoDB
Document JSON complet MongoDB, CouchDB
Colonne Type Big Data Cassandra, HBase
Graphe Relations complexes Neo4j

MongoDB = type Document.

3️⃣ Installer & utiliser MongoDB

Sous Windows :

  1. installer MongoDB Server
  2. créer dossier data/db
  3. lancer mongod (serveur)
  4. utiliser Studio 3T / Mongosh

4️⃣ JSON & BSON

Un document MongoDB = un objet JSON :

{
  "nom": "Amina",
  "filiere": "Dev Digital",
  "notes": [14, 15, 12]
}

MongoDB utilise BSON (JSON binaire) → plus rapide + types supplémentaires.

5️⃣ Modélisation : Embedding vs Linking

🔹 Embedding (imbrication)

Mettre un document dans un autre.

{
  "nom": "Amina",
  "filiere": { "id": "DD", "nom": "Dev Digital" }
}

👍 rapide en lecture
👎 à éviter si les données grossissent

🔹 Linking (référence)

Comme les clés étrangères :

{
  "nom": "Amina",
  "filiere_id": "DD"
}

👍 utile si relations complexes (plusieurs-à-plusieurs)

6️⃣ CRUD avec MongoDB

➤ INSERT

db.stagiaires.insertOne({
  nom: "Amina",
  filiere: "Dev Digital",
  niveau: "2A"
});

➤ FIND

db.stagiaires.find({ niveau: "2A" });

➤ UPDATE

db.stagiaires.updateOne(
  { nom: "Amina" },
  { $set: { niveau: "3A" } }
);

➤ DELETE

db.stagiaires.deleteOne({ nom: "Amina" });

7️⃣ Opérateurs utiles

Comparaison :

  • $gt >
  • $lt <
  • $gte ≥
  • $eq =
  • $ne ≠
  • $in liste

Ex :

db.stagiaires.find({ moy1A: { $gte: 14 } });

8️⃣ Tri & Limite

db.stagiaires.find().sort({ nom: 1 }).limit(5);

9️⃣ Agrégation

Très puissant pour statistiques :

db.stagiaires.aggregate([
  { $group: { _id: "$filiere", moy: { $avg: "$moy1A" } } }
]);

🔟 MongoDB + Python (PyMongo)

Connexion :

from pymongo import MongoClient
client = MongoClient()
db = client.Examples

Requête :

for s in db.Stagiaires.find({"option": "Mobile"}):
    print(s)

1️⃣1️⃣ Import / Export

Export JSON :

mongoexport -d Examples -c Stagiaires -o file.json

Import JSON :

mongoimport -d Examples -c Stagiaires file.json

1️⃣2️⃣ Sécurisation MongoDB

Créer un user :

db.createUser({
  user: "admin",

  roles: [{ role: "userAdmin", db: "Examples" }]
});

S'authentifier :

db.auth("admin", "pass");
