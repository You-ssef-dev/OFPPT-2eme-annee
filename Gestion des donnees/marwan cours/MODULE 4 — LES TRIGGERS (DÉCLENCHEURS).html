Débutant 🟦 PARTIE A — NIVEAU DÉBUTANT

⭐ 1. Qu’est-ce qu’un trigger ?

Un trigger est une procédure stockée qui s’exécute automatiquement quand un événement se produit sur une table.

Les déclencheurs réagissent à :

Événement Avant Après
INSERT BEFORE INSERT AFTER INSERT
UPDATE BEFORE UPDATE AFTER UPDATE
DELETE BEFORE DELETE AFTER DELETE
⭐ 2. Syntaxe simple d’un trigger
SQL
CREATE TRIGGER nom_trigger
BEFORE | AFTER  INSERT | UPDATE | DELETE
ON nom_table
FOR EACH ROW
BEGIN
    -- Code à exécuter automatiquement
END;
⭐ 3. Les pseudo-variables OLD et NEW
Trigger NEW OLD
BEFORE INSERT ✔ valeur insérée ✘
AFTER INSERT ✔ ✘
BEFORE UPDATE ✔ nouvelle valeur ✔ ancienne valeur
AFTER UPDATE ✔ ✔
BEFORE DELETE ✘ ✔ valeur supprimée
AFTER DELETE ✘ ✔

NEW.colonne → nouvelle valeur
OLD.colonne → valeur avant modification

⭐ 4. Exemple simple : empêcher les mineurs d’être ajoutés
SQL
DELIMITER //

CREATE TRIGGER tr_age_min
BEFORE INSERT ON clients
FOR EACH ROW
BEGIN
    IF NEW.date_naissance > DATE_SUB(CURDATE(), INTERVAL 18 YEAR) THEN
        SIGNAL SQLSTATE '50001'
        SET MESSAGE_TEXT = 'Le client doit avoir au moins 18 ans';
    END IF;
END //

DELIMITER ;

➡ Le trigger se déclenche automatiquement avant chaque INSERT.

⭐ 5. Exemple simple : historiser une suppression
SQL
CREATE TRIGGER tr_archive_salaire
BEFORE DELETE ON salaries
FOR EACH ROW
INSERT INTO SalaryArchives(employeeNumber, validFrom, amount)
VALUES (OLD.employeeNumber, OLD.validFrom, OLD.amount);
⭐ 6. Lister et supprimer des triggers

Lister :

SQL
SHOW TRIGGERS;

Supprimer :

SQL
DROP TRIGGER nom_trigger;

Expert 🟪 PARTIE B — NIVEAU EXPERT

⭐ 1. Les vraies raisons d’utiliser un trigger
  • ✔ Contrôles métier (âge, stock minimum, règles d’entreprise)
  • ✔ Historisation (audit log)
  • ✔ Mise à jour automatique d’autres tables
  • ✔ Calculs automatiques
  • ✔ Protection contre des données invalides

Les triggers permettent une cohérence des données garantie, même si plusieurs applications se connectent à MySQL.

⭐ 2. Restrictions importantes
  • ❌ Un seul trigger par type (ex : 1 seul BEFORE INSERT par table)
  • ❌ Impossible de retourner une valeur
  • ❌ Impossible d’utiliser CALL dans un trigger
  • ❌ Impossible de créer des tables temporaires dans un trigger
  • ⚠ Trigger dépend du moteur (InnoDB, MyISAM…)

➡ Pour combiner plusieurs actions, il faut tout mettre dans un seul trigger.

⭐ 3. Les triggers AFTER INSERT avancés

Exemple : mettre à jour une moyenne d’âge après insertion d’un client

SQL
CREATE TRIGGER tr_update_moyenne
AFTER INSERT ON clients
FOR EACH ROW
UPDATE moyenne_age
SET moyenne = (
    SELECT AVG(TIMESTAMPDIFF(YEAR, date_naissance, CURDATE()))
    FROM clients
);
⭐ 4. Triggers UPDATE : détecter les colonnes modifiées
SQL
IF OLD.quantite <> NEW.quantite THEN
    INSERT INTO SalesChanges (salesId, beforeQty, afterQty)
    VALUES (OLD.id, OLD.quantite, NEW.quantite);
END IF;

➡ MySQL ne fait rien automatiquement → c'est au développeur de tester les changements.

⭐ 5. Triggers BEFORE UPDATE : bloquer une modification interdite

Exemple : empêcher qu’un client ne devienne mineur après modification :

SQL
IF NEW.date_naissance > DATE_SUB(CURDATE(), INTERVAL 18 YEAR) THEN
    SIGNAL SQLSTATE '50002'
    SET MESSAGE_TEXT = 'Âge minimum requis';
END IF;
⭐ 6. Détection de fraude ou modifications sensibles

Un trigger peut par exemple :

  • enregistrer l’ancien propriétaire d’un compte
  • loguer toutes les modifications
  • sauvegarder l’adresse avant modification

Exemple :

SQL
CREATE TRIGGER after_sales_update
AFTER UPDATE ON sales
FOR EACH ROW
BEGIN
    IF OLD.amount <> NEW.amount THEN
        INSERT INTO SalesAudit(saleId, oldAmount, newAmount, date_modif)
        VALUES (OLD.id, OLD.amount, NEW.amount, NOW());
    END IF;
END;
⭐ 7. Combiner triggers & contraintes métier

Exemple : bloquer une commande si le stock est insuffisant.

SQL
IF NEW.qte > (SELECT stock
               FROM produits
               WHERE id = NEW.produit_id) THEN
    SIGNAL SQLSTATE '45000'
    SET MESSAGE_TEXT = 'Stock insuffisant';
END IF;
⭐ 8. Trigger BEFORE DELETE + AFTER DELETE

Utile pour :

  • archivage
  • recalcul d’un budget
  • mise à jour des totaux

Exemple :

SQL
CREATE TRIGGER after_salary_delete
AFTER DELETE ON salaries
FOR EACH ROW
UPDATE SalaryBudgets
SET total = total - OLD.salary;
⭐ 9. Séquence professionnelle d’utilisation des triggers

Voici comment un développeur organise ses triggers :

  • BEFORE INSERT → contrôle / validation
  • AFTER INSERT → calculs externes
  • BEFORE UPDATE → interdictions / comparaisons
  • AFTER UPDATE → historisation
  • BEFORE DELETE → archivage
  • AFTER DELETE → mises à jour globales
⭐ 10. Modèle complet de trigger professionnel
SQL
DELIMITER //

CREATE TRIGGER tr_clients_update
BEFORE UPDATE ON clients
FOR EACH ROW
BEGIN
    -- 1. Validation
    IF NEW.date_naissance > DATE_SUB(CURDATE(), INTERVAL 18 YEAR) THEN
        SIGNAL SQLSTATE '45000'
        SET MESSAGE_TEXT = 'Âge minimum 18 ans';
    END IF;

    -- 2. Journalisation
    IF OLD.adresse <> NEW.adresse THEN
        INSERT INTO AuditAdresse(clientId, oldAddr, newAddr, date_modif)
        VALUES (OLD.id, OLD.adresse, NEW.adresse, NOW());
    END IF;
END//

DELIMITER ;
