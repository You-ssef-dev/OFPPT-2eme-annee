Débutant 🟦 PARTIE A — NIVEAU DÉBUTANT

⭐ 1. Pourquoi MySQL possède un langage procédural ?

SQL ne contient pas de structures de contrôle (IF, WHILE…), car ce n’est pas prévu dans la norme. MySQL ajoute donc un langage procédural, permettant de :

✔ créer des blocs logiques
✔ utiliser IF, CASE, WHILE, LOOP, REPEAT
✔ déclarer des variables
✔ gérer les erreurs
✔ manipuler les curseurs
✔ automatiser des tâches complexes

Tout ceci se fait dans :

  • les procédures stockées
  • les fonctions
  • les triggers
  • les événements programmés
⭐ 2. Structure d’un bloc MySQL

Tout code procédural est écrit dans un bloc :

SQL
BEGIN
    -- instructions
END;

On peut imbriquer les blocs :

SQL
BEGIN
   SELECT 'Bloc principal';
   BEGIN
       SELECT 'Bloc interne';
   END;
END;
⭐ 3. Les variables

Déclarer une variable locale :

SQL
DECLARE age INT;
DECLARE nom_client VARCHAR(100);
DECLARE trouvé BOOLEAN DEFAULT FALSE;

Variables de session (externes)
Utilisées hors des procédures :

SQL
SET @max_prix := 100;
SELECT * FROM produit WHERE pu = @max_prix;
⭐ 4. Les structures conditionnelles

✔ IF simple

SQL
IF age < 18 THEN
   SET statut = 'Mineur';
END IF;

✔ IF – ELSE

SQL
IF age >= 18 THEN
   SET statut = 'Adulte';
ELSE
   SET statut = 'Mineur';
END IF;

✔ IF – ELSEIF – ELSE

SQL
IF note >= 16 THEN
   SET mention = 'TB';
ELSEIF note >= 14 THEN
   SET mention = 'B';
ELSEIF note >= 12 THEN
   SET mention = 'AB';
ELSE
   SET mention = 'R';
END IF;
⭐ 5. Les structures CASE

✔ CASE avec sélecteur (comme switch)

SQL
CASE pays
   WHEN 'USA' THEN SET livraison='2 jours';
   WHEN 'Canada' THEN SET livraison='3 jours';
   ELSE SET livraison='5 jours';
END CASE;

✔ CASE sans sélecteur

SQL
CASE
   WHEN quantite > 30 THEN SET msg='Quantité > 30';
   WHEN quantite = 30 THEN SET msg='Quantité = 30';
   ELSE SET msg='Quantité < 30';
END CASE;
⭐ 6. Les boucles (LOOP, WHILE, REPEAT)

✔ 6.1 LOOP (comme "loop infini")

SQL
loop_label: LOOP
   SET i = i + 1;

   IF i = 10 THEN
      LEAVE loop_label;
   END IF;
END LOOP;
Rappels :
→ LEAVE = break
→ ITERATE = continue

✔ 6.2 WHILE

SQL
WHILE v1 > 0 DO
   SET v1 = v1 - 1;
END WHILE;

✔ 6.3 REPEAT (exécuté au moins 1 fois)

SQL
REPEAT
   SET x = x + 1;
UNTIL x > p1
END REPEAT;
⭐ 7. Introduction aux curseurs (cursors)

Un curseur permet de parcourir ligne par ligne un résultat SELECT.

Étapes :

  1. Déclarer le curseur
  2. Ouvrir
  3. Lire avec FETCH
  4. Fermer

Exemple simple :

SQL
DECLARE done INT DEFAULT 0;
DECLARE v_nom VARCHAR(100);

DECLARE cur CURSOR FOR SELECT nom FROM clients;
DECLARE CONTINUE HANDLER FOR NOT FOUND SET done = 1;

OPEN cur;

read_loop: LOOP
    FETCH cur INTO v_nom;
    IF done = 1 THEN
        LEAVE read_loop;
    END IF;
    SELECT v_nom;
END LOOP;

CLOSE cur;
⭐ 8. Gestion d’erreurs (exceptions)

Une erreur peut :

  • provoquer un EXIT (sortie du bloc)
  • ou un CONTINUE (continuer malgré l’erreur)

Exemple :

SQL
DECLARE EXIT HANDLER FOR 1048
SET message = 'Erreur : champ NOT NULL vide';
⭐ 9. Introduction à SIGNAL

Permet de lever une exception personnalisée :

SQL
SIGNAL SQLSTATE '45000'
SET MESSAGE_TEXT = 'Erreur personnalisée';

Expert 🟪 PARTIE B — NIVEAU EXPERT

Maintenant, tu vas apprendre les techniques avancées utilisées dans les vraies applications.

⭐ 1. Portée des variables

Une variable :

  • déclarée dans un bloc parent → disponible dans les blocs enfants
  • déclarée dans un sous-bloc → invisible dans le bloc parent
⭐ 2. Gestion avancée des erreurs

Types de conditions utilisables :

  • SQLWARNING → erreurs commençant par 01
  • NOT FOUND → erreurs commençant par 02 (curseurs)
  • SQLEXCEPTION → toutes les autres erreurs

SQLSTATE spécifique : DECLARE HANDLER FOR SQLSTATE '23000'
Code MySQL : DECLARE HANDLER FOR 1048

⭐ 3. Exceptions nommées
SQL
DECLARE err_required CONDITION FOR SQLSTATE '45000';

DECLARE EXIT HANDLER FOR err_required
SET msg = 'Erreur de validation';
⭐ 4. RESIGNAL (relancer une erreur)

Dans un handler, pour renvoyer l’erreur :

SQL
RESIGNAL SET MESSAGE_TEXT = 'Division par zéro impossible';
⭐ 5. Curseurs avancés — verrouillage FOR UPDATE

Pour modifier les lignes parcourues :

SQL
DECLARE cur CURSOR FOR 
SELECT id, adresse FROM clients FOR UPDATE;
⚠ Attention

FOR UPDATE est interdit si la requête comprend :

  • DISTINCT
  • GROUP BY
  • agrégats
  • UNION
⭐ 6. Combiner curseurs + UPDATE

Exemple réel :

SQL
IF UPPER(v_adresse) = 'AGADIR' THEN
    UPDATE clients
    SET adresse = 'GRANDE AGADIR'
    WHERE id = v_id;
END IF;
⭐ 7. Utilisation professionnelle de LOOP + cursor + exception

C’est le schéma officiel recommandé :

SQL
DECLARE done INT DEFAULT 0;

DECLARE CONTINUE HANDLER FOR NOT FOUND SET done = 1;

OPEN cur;

loop1: LOOP
    FETCH cur INTO v1, v2;
    IF done = 1 THEN LEAVE loop1; END IF;

    -- traitement ici
END LOOP;

CLOSE cur;
⭐ 8. Événements programmés (cron MySQL)

MySQL peut exécuter du code automatiquement :

SQL
CREATE EVENT nettoyage
ON SCHEDULE EVERY 1 DAY
DO
DELETE FROM logs WHERE created_at < NOW() - INTERVAL 30 DAY;
⭐ 9. Exemple complet professionnel

Un code MySQL combinant :

  • variables
  • curseur
  • condition
  • handler d’erreur
  • mise à jour
SQL
CREATE PROCEDURE maj_clients()
BEGIN
DECLARE done INT DEFAULT 0;
DECLARE v_id INT;
DECLARE v_adresse VARCHAR(100);

DECLARE cur CURSOR FOR SELECT id, adresse FROM clients;
DECLARE CONTINUE HANDLER FOR NOT FOUND SET done = 1;

OPEN cur;

boucle: LOOP
    FETCH cur INTO v_id, v_adresse;

    IF done = 1 THEN LEAVE boucle; END IF;

    IF UPPER(v_adresse) = 'CASA' THEN
        UPDATE clients
        SET adresse = 'CASABLANCA'
        WHERE id = v_id;
    END IF;

END LOOP;

CLOSE cur;
END;
