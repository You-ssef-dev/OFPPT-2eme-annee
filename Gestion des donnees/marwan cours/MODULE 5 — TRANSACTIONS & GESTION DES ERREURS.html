Débutant 🟦 PARTIE A — NIVEAU DÉBUTANT

⭐ 1. Qu’est-ce qu’une transaction ?

Une transaction est un ensemble d’instructions SQL qui doivent :

  • ✔ toutes réussir
  • ou ✘ être entièrement annulées.

Ce principe s’appelle ACID :

  • Atomique → tout ou rien
  • Cohérent
  • Isolé
  • Durable

Exemple de transaction simple :

SQL
START TRANSACTION;

UPDATE compte
SET solde = solde - 200
WHERE id = 'ACC1';

UPDATE compte
SET solde = solde + 200
WHERE id = 'ACC2';

COMMIT;   -- valide définitivement

Si un problème apparaît :

SQL
ROLLBACK; -- annule TOUT
⭐ 2. Pourquoi les transactions sont essentielles ?

Exemple : transfert bancaire.

Si le débit fonctionne mais le crédit échoue → argent perdu.

Avec une transaction MySQL :

  • ✔ on garantit que les deux opérations réussissent
  • ✔ ou qu’aucune ne se produit

C’est un élément fondamental de tout système sérieux.

⭐ 3. Syntaxe principale
Instruction Rôle
START TRANSACTION; Démarrer une transaction
COMMIT; Valider définitivement
ROLLBACK; Annuler entièrement

Désactiver la validation automatique :

SQL
SET autocommit = 0;
⭐ 4. Gestion de base des erreurs

MySQL peut détecter certains cas d’erreurs et tu peux les gérer avec des handlers.

Exemple simple :

SQL
DECLARE EXIT HANDLER FOR SQLEXCEPTION
    ROLLBACK;
⭐ 5. Exemple débutant complet
SQL
DELIMITER $$

CREATE PROCEDURE paiement()
BEGIN
    DECLARE EXIT HANDLER FOR SQLEXCEPTION
        ROLLBACK;

    START TRANSACTION;

    UPDATE compte
    SET solde = solde - 100
    WHERE id = 1;

    UPDATE compte
    SET solde = solde + 100
    WHERE id = 2;

    COMMIT;
END $$

DELIMITER ;
  • ✔ Si l’une des deux mises à jour échoue → ROLLBACK automatique
  • ✔ Sinon → COMMIT

Tu sais maintenant :

  • ✔ utiliser START TRANSACTION
  • ✔ valider avec COMMIT et annuler avec ROLLBACK
  • ✔ gérer une erreur simple avec un handler EXIT HANDLER FOR SQLEXCEPTION
  • ✔ pourquoi les transactions sont vitales pour la fiabilité

Expert 🟪 PARTIE B — NIVEAU EXPERT

⭐ 1. Types d’erreurs gérables par MySQL

Dans un handler, tu peux gérer différents types de conditions :

Type Signification
SQLWARNING Avertissements (SQLSTATE '01xxx')
NOT FOUND Aucun résultat (curseur, SQLSTATE '02xxx')
SQLEXCEPTION Toutes les autres erreurs
Code MySQL Numéro d’erreur (ex : 1048)
SQLSTATE 'xxxxx' Identifier précisément une erreur
Exceptions nommées DECLARE CONDITION

Exemple d’erreur NOT NULL :
ERR 1048 : Column ... cannot be null

⭐ 2. Deux comportements possibles des handlers
  • ✔ CONTINUE → L’exécution continue après l’erreur.
  • ✔ EXIT → Le bloc actuel se termine immédiatement.
⭐ 3. Syntaxe générale des handlers
SQL
DECLARE <action> HANDLER FOR <condition>
    <instruction(s)>;

Exemples :

SQL
DECLARE EXIT HANDLER FOR 1048
    SET msg = 'Champ obligatoire !';

DECLARE CONTINUE HANDLER FOR SQLSTATE '23000'
    SET dup = 1;

DECLARE EXIT HANDLER FOR NOT FOUND
    SET finished = 1;
⭐ 4. Exemple professionnel : éviter de planter une procédure
SQL
DECLARE EXIT HANDLER FOR 1048
BEGIN
    SET erreur := 'Un champ NOT NULL est vide';
END;
⭐ 5. Exceptions nommées (CONDITION)

Très utile pour rendre le code lisible.

SQL
DECLARE erreur_age CONDITION FOR SQLSTATE '45000';

DECLARE EXIT HANDLER FOR erreur_age
    SET message = 'Âge invalide';

Tu peux ensuite :

SQL
SIGNAL erreur_age;
⭐ 6. SIGNAL — Lancer une erreur personnalisée

Permet de bloquer une opération avec un message clair.

Exemple :

SQL
IF NEW.quantite <= 0 THEN
    SIGNAL SQLSTATE '45000'
    SET MESSAGE_TEXT = 'La quantité doit être positive';
END IF;
⭐ 7. RESIGNAL — Relancer une erreur depuis un handler

Cas typique :

SQL
DECLARE division_by_zero CONDITION FOR SQLSTATE '22012';

DECLARE CONTINUE HANDLER FOR division_by_zero
    RESIGNAL SET MESSAGE_TEXT = 'Division impossible';
⭐ 8. Gestion avancée d'un curseur avec NOT FOUND
SQL
DECLARE done INT DEFAULT 0;
DECLARE CONTINUE HANDLER FOR NOT FOUND
    SET done = 1;

➡ Obligatoire pour sortir proprement d’une boucle avec curseur.

⭐ 9. Exemple avancé complet (transactions + handlers + conditions)
SQL
DELIMITER $$

CREATE PROCEDURE maj_stock(IN p_id INT, IN p_qte INT)
BEGIN
    DECLARE erreur CONDITION FOR SQLSTATE '45000';

    DECLARE EXIT HANDLER FOR erreur
    BEGIN
        ROLLBACK;
    END;

    START TRANSACTION;

    IF p_qte < 0 THEN
        SIGNAL erreur
            SET MESSAGE_TEXT = 'Quantité invalide';
    END IF;

    UPDATE produits
    SET stock = stock - p_qte
    WHERE id = p_id;

    IF ROW_COUNT() = 0 THEN
        SIGNAL erreur
            SET MESSAGE_TEXT = 'Produit introuvable';
    END IF;

    COMMIT;
END $$

DELIMITER ;

Fonctionnement professionnel :

  • ✔ vérification des paramètres
  • ✔ signaler une erreur propre
  • ✔ rollback automatique
  • ✔ commit uniquement si tout réussit
⭐ 10. Exemple expert : transférer un montant avec sécurité totale
SQL
DELIMITER $$

CREATE PROCEDURE virement(
    IN compte_source INT,
    IN compte_dest INT,
    IN montant DOUBLE
)
BEGIN
    DECLARE EXIT HANDLER FOR SQLEXCEPTION
    BEGIN
        ROLLBACK;
        SIGNAL SQLSTATE '45000'
            SET MESSAGE_TEXT = 'Erreur durant le virement';
    END;

    START TRANSACTION;

    UPDATE comptes
    SET solde = solde - montant
    WHERE id = compte_source;

    UPDATE comptes
    SET solde = solde + montant
    WHERE id = compte_dest;

    COMMIT;
END $$

DELIMITER ;

Ce que garantit ce code :

  • ✔ Aucun argent perdu
  • ✔ Cohérence totale
  • ✔ Aucune opération partielle
